# Instruções do Usuário

- **Edições Manuais Estritas:** Quando o usuário solicitar uma alteração pontual e manual (como alterar uma imagem, um texto, ou usar a ferramenta de edição de código), realize **apenas** essa alteração solicitada. **NÃO** modifique, não refatore e não altere nenhuma outra parte do código que não seja o alvo explícito da solicitação.

- **Envio de E-mails para Novos Artigos:** Toda vez que um novo artigo for criado, um e-mail deve ser enviado para quem entrou na guilda mantendo o formato existente no código. A imagem do artigo deve ser enviada diretamente (do próprio upload, sem ser apenas um link de web), e o botão/link de "ler artigo" deve direcionar o usuário especificamente para a página daquele artigo em questão.
- **Inserção de Imagens Inline no Resend:** Para incluir imagens anexadas no corpo do e-mail com a tag `cid:`, utilize a propriedade `contentId` (com "I" maiúsculo) no objeto de anexo da API do Resend. Use preferencialmente o buffer da imagem (não base64 se estiver corrompendo) e para garantir a entrega correta, dispare os e-mails individualmente através do `resend.emails.send` (agrupados com Promise.all) caso encontre problemas com `resend.batch`.
- **Garantia de Imagem no E-mail:** SEMPRE TEM QUE IR COM A IMAGEM. Ao codificar ou modificar o disparo de e-mails para novos artigos (como em `api/index.ts`), insira um mecanismo de *fallback* para ler `posts.json` buscando o post por id ou utilizar as lógicas de imagens padrão (defaultImages) para que, se nenhuma imagem for passada na request, o e-mail não seja enviado sem foto. Teste a implementação para garantir que a tag `<img src="cid:postimagecid" />` (ou equivalente externa garantida) esteja sendo preenchida sempre.
